BUSGO API - FLOWS CHO CAC ROLE CON LAI
======================================

File nay bo sung cho customer-flow.txt.

Ghi chu chung
-------------
- Role chinh trong auth.user.role:
  - super_admin
  - operator
  - driver
  - customer
- Operator co staff_profile.role:
  - company_admin
  - dispatcher
  - support
- Token la Bearer JWT. Cac endpoint co security bearer can token hop le.
- Login operator/driver/customer dung POST /auth/sign-in.
- Login super-admin dung POST /super-admin/sign-in.
- Route duoc sinh tu file trong src/api.


1. Map quyen theo prefix
------------------------

/super-admin/*
- Can auth.user.role = super_admin.
- Quan ly cong ty, user, company admin, dashboard, promotion news, stripe balance/payout.

/operator-admin/*
- Chu yeu can auth.user.role = operator va staffProfileRole = company_admin.
- Quan ly cong ty cua minh, staff, driver, vehicle, seat, payment, revenue, Stripe Connect.
- Rieng POST /operator-admin/sign-up khong can token; tao request company_admin inactive.

/operator-dispatcher/*
- Can operator + dispatcher hoac company_admin tuy endpoint.
- Quan ly route, station, trip price, trip schedule, stopping point, trip.
- Dispatcher chi co mot so endpoint read driver/vehicle.
- Rieng POST /operator-dispatcher/sign-up khong can token; tao request dispatcher inactive.

/operator-support/*
- Can operator + support hoac company_admin tuy endpoint.
- Quan ly ticket support va coupon.
- Rieng POST /operator-support/sign-up khong can token; tao request support inactive.

/driver/*
- Can auth.user.role = driver, tru POST /driver/sign-up.
- Driver xem trip, route, passenger list, check-in khach, doi trang thai trip, xem stat.

/operator/profile/*
- Can auth.user.role = operator.
- Dung chung cho company_admin, dispatcher, support.

/auth/*
- Phan lon dung chung cho user da dang nhap.
- sign-in, send OTP, reset password, notification, FCM device, social login.


2. Flow Super Admin
-------------------

Super-admin khoi tao va quan tri he thong:

POST /super-admin/sign-in
  -> GET /super-admin/dashboard
  -> POST /super-admin/bus-company
  -> POST /super-admin/company-admin
  -> GET /super-admin/company-admin
  -> PUT /super-admin/company-admin/:userId
  -> GET /super-admin/user
  -> dashboard revenue/user/booking
  -> promotion news
  -> balance / payout


2.1 Auth super-admin
--------------------

POST /super-admin/sign-in
- Login rieng cho super_admin.
- Body:
  {
    email?: string,
    phone?: string,
    password: string
  }
- Business:
  - Tim user theo email/phone.
  - User phai active.
  - User phai co role super_admin.
  - Verify password.
  - Tra token 30 ngay.
- Response:
  {
    message: "Thanh cong",
    token,
    user
  }

-------------------------

- Response:

4. Flow Company Admin / Operator Admin
--------------------------------------

POST /operator-admin/sign-up
  -> super-admin nhan notification verify
  -> super-admin hoac verify endpoint active account
  -> POST /auth/sign-in
  -> GET/PUT /operator-admin/me
  -> POST /operator-admin/stripe/account
  -> tao vehicle + seat
  -> tao route/station/price/schedule qua dispatcher endpoints
  -> quan ly staff/driver/payment/revenue


4.1 Dang ky company admin
-------------------------

POST /operator-admin/sign-up
- Public endpoint.
- Body:
  {
    fullName,
    contactInfo: { email, phone },
    password,
    companyId
  }
- Business:
  - Tao user role operator, status inactive.
  - Tao staff_profile role company_admin, status inactive.
  - Notify super-admin device dau tien.
- Response:
  { message: "Yeu cau tao tai khoan moi da duoc gui den quan tri vien cap cao" }

POST /operator-admin/account/verify
- Can company_admin.
- Body:
  { id: userId, status: active|inactive|banned }
- Business theo code:
  - Update staff_profile.status voi userId va companyId cua admin.
  - Update auth.user.status.
  - Clear driver list cache theo company.
- Ghi chu:
  - Driver sign-up tao company_driver, khong tao staff_profile.
  - Endpoint verify nay update staff_profile truoc, nen flow verify driver co kha nang lech voi code hien tai.

GET /driver/trip/:id/passenger
- Can driver.
- Query:
  {
    limit?: number,
    next?: ticketId,
    phoneNumber?: string
  }
- Business:
  - Chi list passenger cua trip driver dang phu trach.
  - Booking status in [pending, paid].
  - Ticket status in [reserved, paid, checked_in].
  - Co the filter phoneNumber.
- Response:
  {
    passengers: [
      {
        id,
        fullName,
        phoneNumber,
        seatNumber,
        status,
        ticketStatus,
        pickup,
        bookingType,
        totalAmount,
        dropoff
      }
    ],
    next
  }

PUT /driver/trip/:id/passenger/:passengerId/check-in
- Can driver.
- Body:
  {
    status: "reserved" | "paid" | "cancelled" | "checked_in"
  }
- Business:
  - Update ticket theo passengerId va tripId.
  - Neu ticket status = checked_in:
    - booking.status = paid.
    - payment.status = success.
  - Neu ticket status = cancelled:
    - booking.status = cancelled.
    - payment.status = failed.


7.4 Driver update trip status
-----------------------------

PATCH /driver/trip/:id
- Can driver.
- Body:
  {
    status: "scheduled" | "running" | "completed" | "cancelled"
  }
- Business:
  - Update trip.status va set driverId = user hien tai.
  - Neu status = completed:
    - Check con passenger active chua check-in khong.
    - Active passenger la booking pending/paid va ticket reserved/paid.
    - Neu con thi loi TRIP_HAS_UNCHECKED_PASSENGERS.
    - Neu khong con thi tang completedTripCount thang cua departureDate.
  - Neu status = cancelled:
    - Tang cancelledTripCount thang cua departureDate.

GET /driver/me/stat
- Can driver.
- Tra stat thang hien tai, tao row neu chua co.


8. Common Auth APIs
-------------------

POST /auth/sign-in
- Login cho customer/operator/driver, khong cho super_admin qua route nay.
- Body:
  {
    email?: string,
    phone?: string,
    password
  }
- Rate limit: 10 request / 30s.
- User phai active.

POST /auth/send-otp
- Public endpoint.
- Body:
  {
    field: "email" | "phone",
    value: string
  }
- Rate limit: 3 request / 2m.
- Business:
  - OTP cache 120s theo field+value.
  - Luu DB expiresAt = now + 2 phut.
  - Production: email qua email sender, phone qua Infobip.
  - Non-production chi tra message success.

PUT /auth/reset-password
- Public endpoint theo code, schema co security bearer nhung handler khong auth.
- Body:
  {
    otp,
    email?: string,
    phone?: string,
    password
  }
- OTP "555555" bypass.
- Update password theo email hoac phone.

PUT /auth/password
- Can authenticated user.
- Body:
  {
    oldPassword,
    newPassword
  }
- Verify old password roi update password moi.

POST /auth/logout
- Can authenticated user.
- Tang tokenVersion de vo hieu token hien tai/cac token cu.

POST /auth/google/verify-token
- Public endpoint.
- Body:
  { idToken }
- Verify Google token, upsert customer user active.

POST /auth/facebook/verify-token
- Public endpoint.
- Body:
  { accessToken, idToken }
- Verify Facebook token, upsert customer user active.

GET /auth/user
- Can authenticated user bat ky.
- Goi listUsers giong super-admin list user.
- Ghi chu: endpoint nay khong restrict super_admin trong handler, chi requiredAuthenticate.

10. Chat APIs
-------------

POST /chat/box
- Can authenticated user.
- Body:
  {
    receiverId,
    message
  }
- Business:
  - Tao hoac update chat.box theo cap sender/receiver.
  - Tao message dau tien.
  - Tang unread counter cho receiver.
  - Emit websocket event chat:new.

GET /chat/box
- Can authenticated user.
- Query:
  {
    limit: 10..100,
    next?: boxId
  }
- List box cua user, order unreadReceiverCount desc.

POST /chat/box/:id/message
- Can authenticated user.
- Body:
  { message }
- Business:
  - Insert message.
  - Update lastMessage, message counters, unread counters.
  - Emit message:new va chat:unread:count.

GET /chat/box/:id/message
- Can authenticated user.
- Query:
  {
    message?: string,
    limit: 10..100,
    next?: messageId
  }
- Sort newest first; next page dung id < next.

PUT /chat/box/:id/message/:messageId
- Can authenticated user.
- Recall message:
  - chi update message theo id, boxId, senderId trong params user.
  - body thanh "Tin nhan nay da bi thu hoi".

PUT /chat/box/:id/unread-count
- Can authenticated user.
- Mark read:
  - Neu viewer la receiver thi unreadReceiverCount = 0.
  - Neu viewer la sender thi unreadSenderCount = 0.

POST /chat/ai
- Can authenticated user.
- Body:
  { message }
- Rate limit: 30 request / 1m.
- Goi service.openai.chat va tra message.

- Neu user khong co accountStripeId:
  {

  }

- Can stripe-signature va raw body.

- Can header x-cron-secret hop le.
- Business:
  - Lay toi da 200 booking pending co expiredAt < now.

  - pending payment -> failed.
  - ticket != checked_in -> cancelled.
  - xoa seat_segment cua ticket cancelled.

- Can x-cron-secret.
- Business:
  - Tim booking paid/ticket paid hoac cash pending/reserved.
  - Trip scheduled, departureDate la hom nay.
  - Departure >= now + 2 hours.
  - Chua co notification departure-reminder cho booking/trip.
  - Tao notification.

POST /job/cleanup-trip
- Can x-cron-secret.
- Delete trips co departure_date < CURRENT_DATE.
